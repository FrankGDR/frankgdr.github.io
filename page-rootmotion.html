Root motion and mechanim in Unity

Testing Unity's mechanim system with root motion and a little bit of inverse kinematics.

This is a test of Unity's mechanim system for character animations using root motion, which is motion driven by the animations. I started by modelling a robotic humanoid character in Blender and then rigging it.

For the animations, I used Adobe's mixamo.com, which is a free database of animations where you can simply give a fully rigged humanoid character, select the animations you want, and then download a fbx file containing the character with all the animations you chose. Of course, the range of animations to select isn't quite as varied as I would like, but it is still a nice thing to have for someone who isn't familiar with animating characters such as myself.

Once I downloaded all the animations I needed, I started working on the mechanim system. I made a basic node on my state machine for normal locomotion, which expanded into a tree. The animations would blend according to the speed of the character and it's direction. I also added pivoting states for when the player inputs a sudden change in direction.
